## 2차시(1/7) - GitHub Flow
### GitHub Flow 전략
- master branch
- feature branch - 작은 단위의 기능 개발 후 push해 pr 받은 후 병합되면 브랜치 삭제

### GitHub Fork할 때
1. fork 후 clone
2. git remote add upstream으로 상위 레포와 연결
3. git remote -v로 확인하면![[스크린샷 2026-01-07 오전 10.48.36.png]]
4. upstream에 추가된 내용 있으면 현재 내가 수정한 내용 git stash로 가리고 pull해 온 다음 git stash pop해서 충돌나는 부분 resolve
### Git 명령어 사용
- git checkout -b 쓰지 말고 git branch와 git switch 쓰기
- 보고 있는 브랜치 변경시 충돌날 때도 stash로 가리고 이동 후 stash pop하고 git resolve하면 됨
	- 충돌 발생할 경우에는 해결 후에 git stash drop을 수행해야만 함
- .env(api key) 같은 거 공유할 때
	1. private repository 만들고 submodule로 지정
	2. cd로 해당 폴더로 이동해서 수정하고
		1. submodule에서 한 번 commit, push
		2. 바깥 레포지토리에서도 commit, push해야 submodule의 바뀐 해시값을 바라보게 됨
		3. 이후 pr
- reset, reflog, rebase 명령어 익히기
- git push --prune하면 내 로컬 branch들을 날려줌(가장 많이 쓰는 옵션)
